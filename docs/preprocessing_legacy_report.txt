  Task 1: Data Preprocessing — 方案详细说明

一、数据概况
--------------------------------------------------------------------------------
数据集：Student Dropout Dataset
来  源：data/data.csv（分号分隔）
规  模：4424 行 × 37 列（36 个特征 + 1 个 Target）
目  标：Target 列，三分类（Graduate / Dropout / Enrolled）

Target 分布：
  Graduate :  2209 (49.9%)
  Dropout  :  1421 (32.1%)
  Enrolled :   794 (17.9%)
  类别不平衡比 (majority/minority) = 2.78

数据质量：
  缺失值：0（全部 37 列均无缺失）
  重复行：0


二、核心问题诊断
--------------------------------------------------------------------------------
原始数据的 36 个特征全部以数值形式（int64/float64）存储，pandas 的
select_dtypes 会把它们全部识别为 "numeric"。但实际上有大量特征是
整数编码的类别变量（如 Course 列的值为 33, 171, 8014 … 9991，是
课程 ID 编码，数值大小没有实际意义）。

旧方案的问题：
  - 对所有 36 个特征统一做 StandardScaler
  - 类别编码特征（如 Course=9991）被当作连续值标准化，语义错误
  - 没有近零方差过滤，保留了几乎无信息量的特征
  - 没有处理高度相关的冗余特征对


三、新方案：四步预处理流程
================================================================================

【Step 1】数据加载与质量检查
--------------------------------------------------------------------------------
操作：
  1. 读取 CSV（sep=';'），清洗列名空白字符
  2. 分离特征矩阵 X 与目标变量 y
  3. 统计并展示：
     - 数据形状 (4424, 37)
     - Target 各类数量与比例
     - 所有列的缺失值统计（确认全为 0）
     - 重复行检查（确认为 0）
     - 类别不平衡比（2.78:1）

结论：数据质量良好，无需缺失值填充或去重。Pipeline 中仍保留
Imputer 作为防御性措施，确保在新数据上的鲁棒性。


【Step 2】语义特征分类
--------------------------------------------------------------------------------
不依赖 dtype 自动检测，而是根据每个特征的真实物理含义手动分为三类：

A. 二值特征 (Binary) — 8 个
   Daytime/evening attendance    (0/1, 日间/夜间)
   Displaced                     (0/1, 是否搬迁)
   Educational special needs     (0/1, 是否有特殊教育需求)
   Debtor                        (0/1, 是否有欠款)
   Tuition fees up to date       (0/1, 学费是否缴清)
   Gender                        (0/1, 性别)
   Scholarship holder            (0/1, 是否有奖学金)
   International                 (0/1, 是否国际生)

   处理方式：passthrough（直接保留原值，不做变换）

B. 名义类别特征 (Nominal Categorical) — 9 个
   Marital status                (6 个唯一值: 1,2,3,4,5,6)
   Application mode              (18 个唯一值: 1,2,5,7,10,...,57)
   Course                        (17 个唯一值: 33,171,8014,...,9991 — 课程 ID)
   Previous qualification        (17 个唯一值: 1,2,...,43)
   Nacionality                   (21 个唯一值: 1,...,109 — 国籍编码)
   Mother's qualification        (29 个唯一值)
   Father's qualification        (34 个唯一值)
   Mother's occupation           (32 个唯一值)
   Father's occupation           (46 个唯一值)

   处理方式：OneHotEncoder(drop='first')
   drop='first' 是为了避免多重共线性（dummy variable trap），对
   Logistic Regression 等线性模型尤其重要。

C. 数值特征 (Numeric) — 19 个
   Application order             (有序: 0-9, 志愿顺序)
   Previous qualification (grade)(连续: 95-190, 成绩)
   Admission grade               (连续: 95-190, 入学成绩)
   Age at enrollment             (连续: 17-70, 入学年龄)
   Curricular units 1st sem (credited)    (计数)
   Curricular units 1st sem (enrolled)    (计数)
   Curricular units 1st sem (evaluations) (计数)
   Curricular units 1st sem (approved)    (计数)
   Curricular units 1st sem (grade)       (连续: 0-18.88)
   Curricular units 1st sem (without evaluations) (计数)
   Curricular units 2nd sem (credited)    (计数)
   Curricular units 2nd sem (enrolled)    (计数)
   Curricular units 2nd sem (evaluations) (计数)
   Curricular units 2nd sem (approved)    (计数)
   Curricular units 2nd sem (grade)       (连续: 0-18.57)
   Curricular units 2nd sem (without evaluations) (计数)
   Unemployment rate             (连续: 7.6-16.2, 宏观指标)
   Inflation rate                (连续: -0.8-3.7, 宏观指标)
   GDP                           (连续: -4.06-3.51, 宏观指标)

   处理方式：SimpleImputer(median) → StandardScaler()
   标准化理由：原始尺度差异极大，Course 范围 [33, 9991]，成绩范围
   [0, 18.88]，GDP 范围 [-4.06, 3.51]。对距离敏感的算法（t-SNE、
   K-Means、Logistic Regression）必须标准化。


【Step 3】特征过滤
--------------------------------------------------------------------------------

A. 近零方差过滤 (NZV > 95%)
   检测标准：单一值占比超过 95% 的特征
   被删除的特征（3 个）：
     - Educational special needs : 98.8% 的值为 0
     - International             : 97.5% 的值为 0
     - Nacionality               : 97.5% 的值为 1

   理由：这些特征几乎无区分度。特别是对 OneHotEncoding 而言，
   Nacionality 的 21 个类别中 97.5% 属于同一类，展开后会产生
   大量几乎全零的稀疏列，增加噪声而非信息。

B. 高相关冗余过滤 (|r| > 0.90)
   检测到的高相关特征对：
     - 1st sem (credited) ↔ 2nd sem (credited)   : r = 0.945
     - 1st sem (enrolled) ↔ 2nd sem (enrolled)   : r = 0.943
     - Mother's occupation ↔ Father's occupation  : r = 0.910

   被删除的特征（3 个）：
     - Curricular units 1st sem (credited)  — 保留 2nd sem
     - Curricular units 1st sem (enrolled)  — 保留 2nd sem
     - Father's occupation                  — 保留 Mother's

   理由：相关系数超过 0.90 意味着两个特征携带的信息高度重叠，
   保留两者会增加共线性、降低模型可解释性，且不带来额外信息增益。
   保留策略：保留与 Target 相关性更高的一个。

过滤结果：
  原始特征：36 个
  过滤后：  30 个（删除 6 个）


【Step 4】构建 Pipeline 并转换
--------------------------------------------------------------------------------

Pipeline 结构（sklearn ColumnTransformer）：

  ┌─────────────────────────────────────────────────────────────────┐
  │  ColumnTransformer                                             │
  │                                                                │
  │  ┌── num (17 个数值特征) ──────────────────────────────────┐   │
  │  │  SimpleImputer(strategy='median')                       │   │
  │  │  → StandardScaler()                                     │   │
  │  └────────────────────────────────────── 输出: 17 维 ──────┘   │
  │                                                                │
  │  ┌── cat (7 个名义类别特征) ───────────────────────────────┐   │
  │  │  SimpleImputer(strategy='most_frequent')                │   │
  │  │  → OneHotEncoder(drop='first', handle_unknown='ignore') │   │
  │  └────────────────────────────────────── 输出: 146 维 ─────┘   │
  │                                                                │
  │  ┌── bin (6 个二值特征) ───────────────────────────────────┐   │
  │  │  passthrough (不做任何变换)                              │   │
  │  └────────────────────────────────────── 输出: 6 维 ───────┘   │
  │                                                                │
  └───────────────────────────────── 最终输出: (4424, 169) ────────┘

最终矩阵：4424 行 × 169 列
  - 17 维来自数值特征（z-score 标准化后）
  - 146 维来自 7 个类别特征的 OneHot 展开（drop='first' 后）
  - 6 维来自二值特征（原值保留）

完整性验证：
  - 无 NaN 残留 ✓
  - 无 Inf 残留 ✓

六、关键设计决策总结
================================================================================

┌──────────────────────┬─────────────────────────────────────────────────────┐
│ 决策                  │ 理由                                               │
├──────────────────────┼─────────────────────────────────────────────────────┤
│ 手动语义分类          │ dtype 检测把所有特征都判为 numeric，无法区分        │
│ (而非 dtype 自动检测) │ Course=9991 这类 ID 和 Admission grade=127.3       │
├──────────────────────┼─────────────────────────────────────────────────────┤
│ OneHot drop='first'  │ 避免多重共线性，对 LR 等线性模型的系数解释性        │
│                      │ 和数值稳定性至关重要                                │
├──────────────────────┼─────────────────────────────────────────────────────┤
│ Binary passthrough   │ 0/1 特征本身已有明确语义，标准化或 OHE 均无必要     │
├──────────────────────┼─────────────────────────────────────────────────────┤
│ NZV 过滤 (>95%)      │ 98.8% 都是 0 的特征对分类几乎无贡献，且 OHE 后     │
│                      │ 会产生大量稀疏噪声列                                │
├──────────────────────┼─────────────────────────────────────────────────────┤
│ 高相关过滤 (>0.90)   │ 1st/2nd sem credited 相关 0.945，保留两者只增加     │
│                      │ 共线性而不增加信息                                  │
├──────────────────────┼─────────────────────────────────────────────────────┤
│ Pipeline 封装        │ preprocessor 可被 Task 4/5/6 的模型 Pipeline        │
│                      │ 直接复用，确保训练/测试使用相同的变换逻辑            │
├──────────────────────┼─────────────────────────────────────────────────────┤
│ Imputer 保留         │ 虽然当前数据无缺失，但保留 imputer 可保证 pipeline  │
│ (虽无缺失值)         │ 在未来新数据上不会因 NaN 而崩溃                     │
└──────────────────────┴─────────────────────────────────────────────────────┘
